# Shopify Connector — Correctness Verification Report (Bodd Active)

## 1. Bottom line

**Mostly correct, but NOT "running properly" — the live pipeline is frozen and several revenue-truth paths are broken.** The dashboards for Bodd Active (`boddactive-com.myshopify.com`, brand `1a6adb32-…`) DO render real data because the **connect-time backfill** landed 813 orders that flowed cleanly Bronze→Silver→Gold→serving (refreshed today, 2026-06-28 09:49). The foundations are genuinely well-built: HMAC-first verification, brand_id from the DB row (never header/body), deterministic uuidV5 event_ids + Bronze MERGE idempotency, and money as bigint minor-units + sibling `currency_code` with integer-only arithmetic. **However**, live ingestion is **deadlocked** on a stuck `connector_sync_status='syncing'` row with no reaper (frozen 13.5h+), the **live webhook lane is non-functional** (HMAC keyed off a `webhook_secret` the connect flow never stores, and webhook subscriptions are never registered at all), **COD revenue is silently understated** (recognition still reads the retired `gokwik.awb_status.v1`), and **four canonical resource events** (refunds/products/customers/fulfillments) are dropped at the Bronze gate. So: orders display correctly today, but any order state change since 06-27 19:46 (cancellation, refund, COD delivery/RTO) will never reach Brain.

## 4. Recommended actions

### Must-fix before production
1. **Unfreeze ingestion (CRIT).** `apps/stream-worker/src/jobs/shopify-repull/run.ts:442-450` — broaden the CAS predicate to also win when `updated_at < now()-INTERVAL '15 minutes'`, and add a stale-`syncing` reaper to the scheduler (`db/migrations/0112`). Manually clear the Bodd Active row (`state='connected'`) and confirm the next repull lands new Bronze rows.
2. **Fix live webhook HMAC (CRIT).** `ShopifyWebhookStrategy.signatureVerify` (`ShopifyWebhookStrategy.ts:70-77`) — key off the Shopify app `client_secret` (`getShopifyClientSecret()`), OR persist `webhook_secret=client_secret` into the connector secret bundle at connect (`HandleOAuthCallbackCommand.ts:140`). Add an integration test on the real `WebhookPipeline`+`ShopifyWebhookStrategy` path.
3. **Wire webhook registration (CRIT).** Call `RegisterWebhooksCommand.execute()` from `HandleOAuthCallbackCommand` (post instance-save) — also fixes Partner-mandatory compliance webhooks. First make it idempotent (treat 422 as success) and align the topic encoding with the matcher (slash form).
4. **Admit the four resource events (CRIT).** Add `product.upsert.v1`, `customer.upsert.v1`, `refund.recorded.v1`, `fulfillment.recorded.v1` to SERVER_TRUSTED in both `db/iceberg/spark/bronze_materialize.py:106` and `db/iceberg/spark/silver/silver_collector_event.py:76-85`. Add a CI guard: every event_type any silver builder reads must be gate-admitted.
5. **Repoint COD recognition (HIGH).** `silver_order_state.py:248-267` — read `shiprocket.shipment_status.v1` instead of retired `gokwik.awb_status.v1`, via `@brain/logistics-status`. Parity-check delivered-COD revenue.
6. **Switch to offline token (HIGH).** Remove `grant_options[]: per-user` (`InitiateOAuthCommand.ts:73`), or implement online-token refresh.
7. **Stop order-webhook event loss (HIGH).** Drop the transport replay-age gate for order webhooks (`ShopifyWebhookStrategy.ts:279-281`).
8. **Trigger resource backfills (HIGH).** Enqueue per-resource resumable-backfill jobs on connect/onboarding (mirror `RequestConnectorBackfillCommand`).
9. **Close the backfill order orphan (HIGH).** Make `silver_order_state` read `order.backfill.v1` too (or emit `order.live.v1` from backfill) so 24-month history reaches revenue marts.
