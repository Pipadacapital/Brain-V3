**Date:** 2026-07-11 (post-Wave-1 — metrics-server, medallion init, CronWorkflow syncs, and PR #5/#6 git-reconciliation are already live; this report reflects the state AFTER those fixes)
**Scope:** Live prod EKS `brain-prod` (acct 380254378136, ap-south-1) + `infra/terraform/envs/prod` + `infra/helm/*` + `infra/argocd/envs/prod`, verified by 4 parallel read-only dimension agents: (A) IaC/GitOps drift, (B) AWS cost & resource posture, (C) IAM/network/pod security, (D) monitoring & alerting. **ID scheme:** continues the Phase-1 `AUD-*` register with the `AUD-INFRA-*` prefix. Overlaps with Phase-1 findings are cross-referenced, not re-counted.

**Phase-1 closures confirmed this round:** AUD-LIVE-1 (metrics-server live, all 5 HPAs resolving), AUD-LIVE-4 partially (Connect landing runs; but see AUD-INFRA-023 — it is still metric-blind), AUD-SEC-1 (EKS API back to pinned /32, committed). ## 0. Executive summary

Prod is in genuinely strong shape after Wave-1: git matches applied AWS state (refresh-only drift is representation noise only), all 16 charts have `values-prod.yaml`, every running service maps to a digest-pinned ArgoCD app, the AppProject allowlist matches live, IRSA trust policies are NN-3-correct with tightly scoped per-workload policies, S3 posture (KMS, TLS-deny, WORM scoped to audit bucket only) is exemplary, and metrics-server/HPAs/Prometheus targets are all healthy.

Four problems are decision-critical:

1. **The next routine `terraform apply` will delete the Redis-from-EKS SG rule** (inline-vs-standalone rule conflict) — a guaranteed prod cache outage sitting in an otherwise-clean plan (AUD-INFRA-001).
2. **Alertmanager routes everything to the null receiver** — 30+ Brain SLO rules and two currently-firing warnings reach no human (AUD-INFRA-022).
3. **The Strimzi Kafka broker data PVCs are live ArgoCD prune candidates** — the committed guard verifiably never propagated to the cluster (AUD-INFRA-002).
4. **The data spine is detection-blind**: the sole Bronze writer (Kafka Connect, ADR-0010) emits zero metrics, the pipeline-stall alert keys on a metric deleted at the cutover, and Trino/freshness are unscraped (AUD-INFRA-023/024/025).

Secondary theme: estimated run-rate (~$580–640/mo) exceeds the $500/mo target while Cost Explorer reads $0.00 — the only cost guardrail currently provides zero signal.
