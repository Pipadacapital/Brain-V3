# ADR-0006 P4 — Connector raw-emit + PySpark-Silver normalization: design pack

_Generated by the `p4-connector-raw-normalize` multi-agent workflow (9 agents, 8 connector designs + synthesis). The blueprint for moving every connector OFF pre-normalized canonical events and ONTO raw provider payloads normalized in Spark Silver (ADR-0006 P4)._

## Master integration plan

### Exemplar Connector
SHOPIFY orders (apps/stream-worker/src/jobs/shopify-repull/run.ts + apps/core/.../strategies/ShopifyWebhookStrategy.ts → db/iceberg/spark/silver/silver_shopify_order_normalize.py). Rationale: (a) ADR-0006 P4 explicitly names Shopify-first then Woo; (b) it is the only candidate that exercises EVERY shared primitive at full surface — strict + resilient money ports, line-item child fan-out, hash_identifier email AND phone (forcing the salt-in-Spark keystone that 6 of 8 connectors depend on), the uuid_shaped event_id, occurred_at ms-precision, the projectOrderStitch operational side-effect that must stay imported, and the additive WebhookPipeline rawPassthrough — so proving the framework here de-risks the maximum downstream surface; (c) it writes into the existing server-trusted silver_collector_event lane so silver_order_state/line/cod_rto need ZERO change, making the parity diff clean; and (d) WooCommerce immediately follows on the IDENTICAL canonical order.live.v1, validating the multi-source-unification seam (silver_order_event) as the very next step. The heaviness (M-L, ~4-6d) is the point — the framework absorbs the reusable parts so connectors 2-8 are mostly field-mapping.

### Integration Order
- 1. SHOPIFY orders (exemplar) — ADR-0006 P4 names it first; it forces EVERY shared primitive at full surface (strict+resilient money, line-item fan-out, hash_identifier email+phone via the salt-in-Spark keystone, uuid_shaped event_id, projectOrderStitch side-effect that must stay imported). Building the framework here proves the keystone (salt-in-Spark) that 6 of 8 connectors depend on. Writes into the server-trusted silver_collector_event lane so silver_order_state/line/cod_rto need ZERO change.
- 2. WOOCOMMERCE orders — emits the IDENTICAL canonical order.live.v1, so it immediately validates the multi-source-unification seam: introduce/confirm the source-agnostic silver_order_event union table and source-scope the silver_order_state/line repoint so Shopify (still on collector lane until its cutover) is never double-counted. Reuses Shopify's salt path, money ports, uuid port verbatim.
- 3. AD-SPEND (Meta + Google) — second multi-source unification (two platforms→one silver_marketing_spend via shared _normalize_spend.normalize, envelope.source discriminates), but NO contact PII, so it isolates the money-port + currency-authority-in-envelope work from the salt path. All three target marts (silver_marketing_spend/campaign/ad_account) are already registered in mart_registry.py; populates silver_ad_account for the first time (its parity SKIPs — verify out-of-band).
- 4. GA4 sessions — single flat grain, no PII, but the ODD parity case (ga4.session.v1 is dropped by the R2/R3 pixel gate today → no live canonical baseline), so it must use golden-fixture parity or a temporary SERVER_TRUSTED entry. Do it after the harness is mature so the fixture-baseline mode is well-trodden. Also surfaces that GA4 ingestion may be silently broken in V4 (separate fix).
- 5. SHIPROCKET shipments — logistics-status classify port (3 frozen label sets) + AWB hash via hash_salted_bytes + shares silver_shipment_event with gokwik; no money column (parity analogue = terminal_class multiset). Scope leg/parity to source='shiprocket'.
- 6. GOKWIK (3 lanes: awb_status + rto_predict + webhook) — the bridge connector: it owns the OTHER leg of silver_shipment_event (awb, shared with shiprocket) AND the OTHER leg of silver_checkout_signal (rto_predict, shared with shopflo). Doing it adjacent to both shared-mart neighbours lets each shared mart be cut over once both legs pass parity.
- 7. SHOPFLO checkout — checkout_abandoned; shares silver_checkout_signal with gokwik's rto_predict leg (carve out ONLY source='shopflo'). Reuses gokwik/shiprocket money+hash+uuid patterns; also retire the legacy duplicate shopfloWebhookHandler.ts.
- 8. RAZORPAY settlement — LAST, deliberately: most compliance-loaded (PCI SAQ-A card.* boundary now lands raw in Bronze), 5 entity-type discrimination, hash_salted_bytes for payment_id/utr/refund/dispute ids, and zero current Bronze rows (empty-vs-empty parity is not evidence → needs seeded/replayed fixtures). Do it only when the framework, the salt path, and the Security-Reviewer D4/PCI sign-off process are all battle-tested.

### Shared Framework
Build ONE reusable PySpark module — db/iceberg/spark/silver/_raw_normalize.py (sibling to the existing iceberg_base.py / _silver_base.py) — so each connector's normalizer collapses to just its field-mapping. It exports the primitives every design re-implements:

(1) IDENTITY — `uuid_shaped(seed_col)` as a pure column expression (sha2→first 16 bytes→version nibble forced 0x5 at hex[12]→variant nibble (n&3)|8 at hex[16]→8-4-4-4-12), PLUS a byte-identical Python-UDF `uuid_shaped_udf` used only by golden-vector tests; a CI unit test asserts the two agree and both equal Node hashToUuidShaped (packages/connector-core/src/util/hash-to-uuid-shaped.ts). This is shared by ALL eight connectors.

(2) MONEY (I-S07, integer-only, never float) — a family covering every observed variant: `decimal_to_minor_strict(col)` (regex ^\d+(\.\d{1,2})?$ guarded, malformed→NULL→quarantine, mirrors decimalStringToMinor that THROWS), `decimal_to_minor_resilient(col)` (null/0 for optional depth fields, tryDecimalToMinor), `major_to_minor_trunc(col)` (pad-then-substr frac to exactly 2, TRUNCATE not round — Meta/GA4/Shopflo), `micros_to_minor(col)` (BigInt div 10000 — Google), `paisa_passthrough(col)` (cast bigint, regex ^\d+$ — Razorpay). NEVER routes through double. Every money column is emitted with a sibling currency_code; currencies are never blended.

(3) PII — TWO DISTINCT hashing conventions centralized as separately-named fns so they are never confused: `hash_identifier(value, salt_hex, kind)` = sha256( utf8(salt_hex) || '||' || normalized ) [salt as STRING, '||' separator — email/phone/Shopify/Woo/Shopflo] and `hash_salted_bytes(value, salt_hex)` = sha256( bytes.fromhex(salt_hex) ++ utf8(lower(trim(value))) ) [salt as HEX BYTES, NO separator — AWB/Razorpay-id/UTR]. Plus `normalize_phone_in(col)` (E.164 IN rules: strip to [\d+], 10-digit→+91…, 0+10→+91+last10, +91+10 passthrough, else digit-stripped fallthrough).

(4) SALT SOURCE (the keystone) — `load_brand_salts(spark)`: ONE JDBC read of a per-brand salt dimension returning (brand_id, salt_hex), broadcast-joined by brand. Centralizes the KMS-unwrap concern: a single SECURITY-DEFINER set-returning fn get_brand_identity_salt_all() (or a pre-unwrapped secure view) over tenancy.brand_identity_salt (migration 0109). Confirm the ONE SoR table/column (the designs name five candidates — identity.brand_identity_salt / tenancy.brand_identity_salt_hex_v / identity.brand_salt / identity.brand_hash_salt / get_brand_identity_salt_all — pick one) and that dev-derived (resolveDevSaltHex = sha256('brain-dev-identity-salt-v1||'+lower(brand_id))) vs prod KMS-unwrapped resolve to the SAME hex the connector used.

(5) TIME — `occurred_at_iso(col, *fallbacks)` reproducing toUtcIso/statDateToIso (append 'Z' to gmt-naive, ms-precise so event_id seeds match JS Date.toISOString()).

(6) ENVELOPE + MERGE — `read_raw(spark, table)` selects brand_id ONLY from the server-trusted envelope column (MT-1) and the verbatim body as a JSON string under `raw`/`provider`; `merge_on_pk(spark, fqtn, df, keys=['brand_id','event_id'], order_by_desc=['ingested_at','occurred_at'])` does the replay-safe row_number dedup + Iceberg MERGE. create_iceberg_table already exists in iceberg_base. With this module a connector normalizer is ~40 lines of get_json_object field mapping (see the ad-spend _normalize_spend.py and ga4 silver_ga4_session.py drafts as the canonical shape).

### Parity Harness
ONE reusable dual-run harness extending the existing db/iceberg/spark/parity/parity_oracle.py + mart_registry.py, parameterized by connector so any lane plugs in. Mechanics:

DUAL-RUN: keep the canonical path live (connector→collector.event.v1→silver_collector_event→silver marts) and build the raw→normalize path into a SHADOW namespace (conventionally *_fromraw / *_shadow / *_srnorm / brain_silver_p4). For shared marts, SOURCE-SCOPE both sides (WHERE source='X'). Register each shadow mart as a MartSpec(name, pk, money_columns) in mart_registry.py.

The parameterized checks — all must return ZERO rows / zero delta (ROW_TOLERANCE=0, MONEY_TOLERANCE=0, the oracle defaults):
(1) KEY parity — symmetric LEFT ANTI JOIN on the PK (e.g. brand_id,order_id / brand_id,event_id): no missing_in_fromraw, no extra_in_fromraw.
(2) MONEY parity — per (brand_id, currency_code) Σ of each money column, delta MUST be 0, FULL OUTER JOIN so neither side hides rows; currencies never blended. For NO-money lanes (shiprocket, gokwik) the analogue is the terminal_class / classification MULTISET (it drives the bigint-minor cod_rto_clawback downstream).
(3) IDENTITY/FIELD parity — per matched key, null-safe (<=> / IS DISTINCT FROM) compare of hashed PII (hashed_customer_email/phone, awb_number_hash, payment_id_hash…), discriminators (entity_type/level/platform), lifecycle (lifecycle_state/is_terminal/terminal_class/dispute_direction), brain_id, currency_code. THE load-bearing check — proves the salt||sha256 port AND the field mapping byte-match.
(4) EVENT-LEVEL hashed-PII parity at source grain — JOIN silver_collector_event (canonical) to the shadow on (brand_id,event_id), compare get_json_object(payload,'$.properties.hashed_customer_email/phone) → proves the PySpark hash == TS hashIdentifier.
(5) CHILD parity where applicable — silver_order_line Σ line_total_minor per (brand,order,currency), index-label-tolerant.
Plus a NEGATIVE CONTROL (perturb one field → assert the check turns RED, non-tautology) and an ISOLATION check (every shadow brand_id ∈ the connector's enumerated brand set — no brand leaked from a body).

GOLDEN-VECTOR UNIT tier (CI, no cluster): drive each retired TS mapper's EXISTING fixtures/tests (e.g. packages/ga4-mapper/src/index.test.ts cases "12.34"→1234/"12"→1200/"12.3"→1230; shopify/woo line-item fixtures; razorpay recon+webhook fixtures) through the normalizer.build() on a tiny raw fixture and assert byte-equality on event_id + each money column + each hash. The TS mapper test suites BECOME the parity golden fixtures.

GATE: every check returns 0 across ≥2 dual-run cycles AND golden-vector units pass AND quarantine tables are empty for the window → open cutover. SPECIAL CASES baked in: GA4 has no live canonical baseline (use golden fixtures or a temporary SERVER_TRUSTED add for the dual-run window); Razorpay has zero current Bronze rows (empty-vs-empty is NOT evidence — require seeded/replayed dual-run before declaring parity); ad-spend silver_ad_account is a NEW row set (parity SKIPs current-mart-absent — verify out-of-band).

### Common Gotchas
- EVENT_ID byte-parity: uuid_shaped must reproduce hashToUuidShaped exactly (sha256→16 bytes→version 0x50→RFC-4122 variant 0x80→8-4-4-4-12) AND the seed STRING must be byte-identical including the missing-key fallbacks ('unknown' vs '' for absent AWB in shiprocket; request_id??'' in gokwik rto; each GA4 dim coalesced to ''). The updated_ms/occurred_at must equal JS Date(...).getTime()/toISOString() to ms — Spark unix_timestamp loses sub-second, so use a ms-precise extraction. LOWER-RISK POLICY: where the connector already computes event_id for Redis/webhook dedup (shopflo, razorpay, gokwik webhook), CARRY it in the server-trusted envelope and let Spark trust it — eliminates re-derivation. Otherwise port + a golden-vector unit test (Spark==TS) before any dual-run.
- MONEY integer-only: never cast totalRevenue/total/spend through a double. Match TS strictness EXACTLY — strict core-amount fields THROW in TS → in Spark route the row to a quarantine (shopflo silver_shopflo_checkout_quarantine) or NULL+filter, NEVER silent 0; resilient depth fields use the null/0 variant. KWD/JPY (≠2 decimals): the port must reproduce the TS mapper's (wrong) 2-dp truncation for parity, flagged for the multi-currency follow-up. A strictness mismatch shows up as a money Σ delta.
- TWO hashing conventions — the silent identity-breaker: email/phone use hash_identifier (salt as STRING + '||' separator); AWB/Razorpay-id/UTR use hash_salted_bytes (salt as HEX BYTES, no separator). Confusing them passes type-check but breaks every brain_id join and parity check #3/#4. Keep them as separately-named fns and golden-vector each.
- SALT-IN-SPARK keystone (shopify/woo/gokwik/shiprocket/shopflo/razorpay): PII hashing moved off the connector boundary INTO Silver, so Spark must read the per-brand salt over JDBC. Dev salt is deterministic; PROD salt is KMS-WRAPPED — needs a KMS-unwrap fn/view. If Spark-resolved salt != the connector's salt for a brand, every hashed_customer_email/awb_number_hash silently diverges. Build/verify ONE salt dimension before the exemplar; fail-loud on a missing salt for any brand that has rows in the lane.
- BRAND AUTHORITY (MT-1): brand_id is read ONLY from the server-trusted envelope top-level column (set from the SECURITY-DEFINER enumerate fn / HMAC-resolved ConnectorInstance), NEVER from raw/provider body. Normalizers use F.col('brand_id'), never get_json_object(raw,'$.brand'). Add a naming-guard/lint assertion. RELATED currency authority: Meta account_currency/account_timezone come from fetchAccountMeta (server context), NOT the Insights row — they MUST be stamped into the envelope or Bronze loses currency authority and money becomes uncurrencied (ad-spend's single highest-risk emit change).
- MULTI-SOURCE UNIFICATION — one canonical entity, many per-source normalizers feed one Silver table: order.live.v1 (shopify+woo via source-agnostic silver_order_event), spend.live.v1 (meta+google via shared _normalize_spend into silver_marketing_spend), silver_shipment_event (shiprocket+gokwik-awb), silver_checkout_signal (gokwik-rto + shopflo). RULE: keep the canonical schema source-agnostic; SOURCE-SCOPE every repoint/leg-removal/parity query (WHERE source='X') so the not-yet-migrated leg is untouched and never double-counted. Disjoint Bronze field unions (date_start vs segments_date) rely on Connect evolve-schema → a field typo silently NULLs→0; the field-parity SQL is the only guard.
- D4 raw-PII in Bronze (gates EVERY prod flip): raw email/phone/AWB/card now transiently land un-hashed in brain_bronze.*_raw for the first time. Mandatory 7-day TTL + snapshot-expiry + RTBF/erasure coverage on the raw namespace + Security-Reviewer sign-off. Razorpay is worst — card.* is a PCI SAQ-A scope EXPANSION; recommend STILL stripping card.* at emit ('raw-minus-PAN'). Shopflo/Razorpay-webhook/Gokwik-webhook keep email/phone boundary-hashed as the exception where event_id never seeds on them (parity-neutral, keeps worst PII out of Bronze).
- DEDUP moved to Silver: Connect sink is at-least-once/append → duplicate raw rows are EXPECTED. merge_on_pk (brand_id,event_id) latest-wins is the new dedup SoR — verify the legacy in-stream Redis / Bronze-MERGE dedup is fully RETIRED for the lane or shadow counts double. line_index can swap on exact-tie lines → compare Σ per (order,currency), not the index label.
- SHARED WEBHOOK PIPELINE blast radius: WebhookPipeline produces the canonical envelope to the collector topic; routing raw needs an ADDITIVE rawPassthrough/rawTopicOverride on PayloadMapResult/IWebhookStrategy — keep additive so the co-implementing Woo/gokwik/shiprocket/shopflo/razorpay strategies are unaffected until their own cutover. ALSO: raw emission defers normalization to the Silver batch (v4-refresh-loop), so live order/shipment freshness now follows the Silver cadence not the stream — confirm the SLA. ALSO: retire duplicate/legacy emit paths (shopflo's standalone shopfloWebhookHandler.ts) and drop the lane from SERVER_TRUSTED (silver_collector_event.py, bronze_materialize.py) + bronzeBridges.ts at cutover, or Bronze mixes canonical+raw.

### Emit Path Changes
Per connector, 1–3 emit sites change identically: the trailing-window re-pull job(s) under apps/stream-worker/src/jobs/*/run.ts (shopify-repull, woocommerce-orders-repull, meta-spend-repull + google-ads-spend-repull, ga4-repull, gokwik-awb-repull + gokwik-rto-predict-emit, shiprocket-shipment-repull, razorpay-settlement-repull) AND the webhook strategy under apps/core/src/modules/connector/webhooks/strategies/*WebhookStrategy.ts (Shopify/WooCommerce/Gokwik/Shiprocket/Shopflo/Razorpay).

AT EACH SITE: (a) DELETE the map*ToEvent(...) call + the uuidV5From*(...) seed + the CollectorEventV1Schema.parse({event_name, properties}) canonical-envelope build, and STOP producing to LIVE_TOPIC. (b) REMOVE the SaltProvider/saltHex fetch — PII hashing leaves the connector and moves to Silver — EXCEPT the D4 boundary-hash exception (KEEP email_hash/phone_hash via hashIdentifier in the Shopflo, Razorpay, and Gokwik WEBHOOK strategies, since their event_id never seeds on those identifiers, keeping the worst PII out of Bronze at zero parity cost; strip raw email/phone from the body before emit). (c) BUILD the thin server-trusted envelope: { brand_id (MT-1 — from the SECURITY-DEFINER enumerate fn result on repull / the HMAC-resolved ConnectorInstance on webhook, NEVER the provider body), source, resource, connector_instance_id, fetched_at, + server-context fields that the body lacks: account_currency/account_timezone (ad-spend Meta), property_id/currency_code/sampling (ga4), data_source ('real'|'synthetic'), and optionally the server-computed event_id where dedup already needs it } with the verbatim provider object nested under raw/provider/order. (d) KEEP buildPartitionKey(brandId, naturalKey) brand-first, injectKafkaTraceContext, cursor/lock/sync_status, HMAC verify + appid→brand resolver. (e) Change LIVE_TOPIC → RAW_TOPIC = {env}.{connector}.raw.v1.

PRESERVE these operational side-effects in TS even though the mapper is dropped (they read raw fields the connector already holds, are PG operational writes not Bronze normalization): Shopify projectOrderStitch → connector_journey_stitch_map (must stay IMPORTED), Razorpay payment.captured → connector_razorpay_order_map (MB-1).

SHARED PLUMBING: packages/contracts/src/events — one generic RawProviderEnvelopeSchema (Zod, raw as passthrough) + a {X}_RAW_V1_TOPIC_SUFFIX const per connector. apps/core .../webhooks/platform/WebhookPipeline.ts + IWebhookStrategy.ts — add an ADDITIVE rawPassthrough/rawTopicOverride so the shared pipeline routes raw to the raw topic without touching un-migrated strategies. infra/kafka-connect — one iceberg-bronze-{connector}.json per connector, cloned from iceberg-bronze-collector.json: topics={env}.{X}.raw.v1, iceberg.tables=brain_bronze.{X}_raw, auto-create + evolve-schema + schema-force-optional, JsonConverter schemas.enable=false, partition bucket(N, brand_id) + days(_kafka_timestamp). At CUTOVER drop the lane's event_type from SERVER_TRUSTED (db/iceberg/spark/silver/silver_collector_event.py, bronze_materialize.py) and remove its bronzeBridges.ts entry.

### Rollout Rollback
Per ADR-0006 every phase is independently shippable + reversible behind a dual-run, with NO data loss because the canonical path stays LIVE until cutover.

FORWARD (per connector): (1) land the shared framework + the ONE salt dimension (KMS-unwrap fn/view) — done with the exemplar. (2) Ship the emit-raw change behind a per-connector flag: raw topic + iceberg-bronze-{X}.json Connect sink + Bronze table, with the connector STILL emitting canonical on the collector lane (or the new raw lane writing into a SHADOW namespace) so both paths run. (3) Prove raw lands in brain_bronze.{X}_raw WITHOUT Spark (Connect-only). (4) Run the normalizer into SHADOW; run the parity harness ≥2 dual-run cycles + golden-vector unit tests to 0 diff; iterate on salt resolution + ms-precision + strictness divergences. (5) Security-Reviewer D4 sign-off on raw-namespace 7-day TTL + RTBF (and PCI card.* posture for Razorpay) — EXTERNAL-BLOCKING for prod. (6) Cutover: point Silver readers at the raw-derived canonical rows / source-scoped union, flip the strategy+repull to the raw topic, drop the lane from SERVER_TRUSTED + bronzeBridges, retire the TS mapper. Refresh via the existing tools/dev/v4-refresh-loop.sh.

ROLLBACK (per connector, a config flip): repoint Silver back at silver_collector_event (canonical lane), re-enable the TS mapper emit on the collector topic, re-add the SERVER_TRUSTED entry + the bronzeBridge. Because dual-run kept the canonical path live and the shadow was separate, rollback is non-destructive; the raw Bronze short-retention bounds the PII-exposure window even if rolled back.

SEQUENCING GATES: each SHARED-MART cluster (order: shopify+woo; shipment: shiprocket+gokwik; checkout: gokwik+shopflo; spend: meta+google) must have BOTH legs pass source-scoped parity before the union/scoping is collapsed, or risk double-count. Razorpay's prod flip is externally blocked on Security/PCI sign-off; GA4 and Razorpay additionally require fixture/seeded baselines (no live canonical to diff). Naming-guard CI (tools/lint/v4-naming-guard.sh) must pass on the new raw tables/topics/shadow tables before each merge.

## Per-connector designs

### shopify
**canonical_events**:
- order.live.v1 — the SINGLE canonical event this connector emits today (constant ORDER_LIVE_V1_EVENT_NAME in packages/shopify-mapper/src/event-names.ts). Emitted by both the webhook (ShopifyWebhookStrategy.payloadMap order branch) and the live re-pull (shopify-repull/run.ts). Its properties payload = OrderProperties (packages/shopify-mapper/src/resources.ts has the resource events; index.ts defines OrderProperties).
- Out of P4 scope but same connector family (separate raw lanes in later sub-phases): order.backfill.v1 (uuidV5FromOrderBackfill), product.upsert.v1, customer.upsert.v1, refund.recorded.v1, fulfillment.recorded.v1 (resources.ts). silver_fulfillment reads fulfillment.recorded.v1 and silver_refund reads refund.recorded.v1 — they are NOT fed by order.live.v1, so flipping order.live.v1 alone does not move them; they flip when shopify.fulfillments.raw / shopify.refunds.raw land.

**raw_topic**:
{env}.shopify.orders.raw.v1  (env = 'prod' when NODE_ENV=production else 'dev', identical prefixing to the existing {env}.collector.event.v1 lane; topic prefix is decoupled to NODE_ENV per the two-env model). Materialized by Kafka Connect Iceberg Sink → brain_bronze.shopify_orders_raw (truly raw, append-only, short retention per ADR-0006 D4).

**raw_provider_shape**:
Kafka value (JsonConverter, schemas.enable=false) = the server-trusted envelope with the verbatim Shopify order nested under `order`: { brand_id (server-trusted, MT-1), source:'shopify', resource:'orders', connector_instance_id, fetched_at, order: { id, name, currency, current_total_price ('decimal string'), financial_status, fulfillment_status, gateway, payment_gateway_names[], cancelled_at, created_at, processed_at, updated_at, customer:{id,email(RAW),phone(RAW)}, note_attributes[{name,value}], line_items[{id,sku,title,name,quantity,price,product_id,variant_id,total_discount}], tax_lines[{title,rate,price}], total_tax, shipping_lines[{title,price}], total_discounts, discount_codes[{code,amount,type}], refunds[{id,created_at,processed_at,note,transactions[{amount,kind,status}]}] } }. The Iceberg sink explodes this into a struct schema; the normalizer reads order.* via the nested struct (or get_json_object on a rebuilt JSON, mirroring silver_collector_event.py's to_json(struct(*cols)) reconstruction). NOTE (D4): raw customer.email / customer.phone land UN-hashed in Bronze for the first time — short-retention + RTBF coverage on brain_bronze.shopify_orders_raw is mandatory and gates the prod flip (Security-Reviewer sign-off).

**ts_mapper_summary**:
@brain/shopify-mapper::mapOrderToEvent(order, saltHex, regionCode='IN', 'order.live.v1') → { event_name, occurred_at, properties: OrderProperties }. Faithful normalization to port: (1) occurred_at = ISO(updated_at ?? processed_at ?? created_at) for live. (2) MONEY (I-S07, integer-only, NEVER float): decimalStringToMinor strictly validates /^\\d+(\\.\\d{1,2})?$/ then computes whole*100 + fracPadded2 as BigInt — amount_minor = decimalStringToMinor(current_total_price), currency_code = order.currency (sibling, never blended). **risks**:
- PII re-hash parity is the keystone risk: the Spark salt-UDF must reproduce hashIdentifier byte-for-byte (sha256hex(salt||'||'||normalized), email=trim+lower, phone=normalizePhone IN-rules). ANY drift breaks identity joins (brain_id) AND parity-oracle check (4). The per-brand salt lives KMS-WRAPPED in tenancy.brand_identity_salt (get_brand_identity_salt, migration 0109) — Spark cannot read raw hex without a KMS-unwrap step. Need a salt_hex dimension view/UDF (KMS decrypt in the job or a pre-unwrapped secure dim). In dev the salt is the deterministic resolveSaltHex value; prod is the hard case.
- D4 compliance: raw Woo order JSON contains raw billing.email/phone → non-consented raw PII now transiently lands in brain_bronze.woocommerce_orders_raw. Requires the ADR-0006 D4 short-retention TTL (7d) + RTBF/erase coverage on the raw namespace + Security-Reviewer sign-off before the prod flip. Storefront orders are a server-trusted lane (no R3 consent gate), so the gate is retention, not admission.
- Phone normalization fidelity: normalizePhone has IN-specific branches + a low-confidence fallthrough (returns digit-stripped form). The PySpark port must reproduce the SAME fallthrough (not throw, not drop) or hashes diverge for non-IN/edge numbers.
- uuid_v5 event_id parity: the Python UDF must reproduce hashToUuidShaped (sha256→16 bytes→version 0x5→RFC4122 variant) AND the updatedAtMs seed must equal the TS Date.parse(occurred_at) ms. occurred_at UTC-coercion of gmt-naive timestamps (append 'Z') must match toUtcIso exactly or the ms (and thus event_id) drift and dedup/parity break.
- Money edge cases: tryDecimalToMinor returns null (→ treated as 0) on malformed optional fields; decimalStringToMinor THROWS on the core amount (negative total rejected). The Spark UDF returns None on malformed — the order-level amount must still fail-loud or be quarantined, not silently 0, to preserve the 'never blend/never lose money' rule.
- Source repoint blast radius: silver_order_state and silver_order_line currently read silver_collector_event; adding silver_order_event as a UNION source must not double-count Shopify orders (still on the collector lane until ITS P4). Scope the union/repoint to source='woocommerce' during the staged per-source cutover.
- Multi-source unification: Shopify and Woo emit IDENTICAL order.live.v1 today; after P4 each has its own raw topic + normalizer. The canonical silver_order_event schema must stay source-agnostic so Shopify's later port writes the same columns (preserve the 'one canonical entity, many per-source normalizers' ADR contract).
- Webhook latency: today the webhook strategy maps + emits canonical synchronously; raw emission defers normalization to the Silver batch, so live order state freshness now depends on the Silver job cadence (v4-refresh-loop) rather than the stream. from __future__ import annotations

# ── toCountString: integer part of "123"/"123.0"; null/''/invalid → NULL ───────────────
def _count(c):
    s = F.trim(c.cast("string"))
    return F.when(s.rlike(r'^\d+(\.\d+)?$'),
                  F.regexp_extract(s, r'^(\d+)', 1).cast("bigint")).otherwise(F.lit(None).cast("bigint"))

# ── majorDecimalToMinorString: BIGINT minor units, integer-only, TRUNCATE frac to 2 (I-S07) ──
def _money_minor(c):
    s = F.trim(c.cast("string"))
    whole = F.regexp_extract(s, r'^(\d+)', 1)
    frac_raw = F.regexp_extract(s, r'^\d+\.(\d+)$', 1)              # '' when no fractional part
    frac2 = F.substring(F.rpad(frac_raw, 2, '0'), 1, 2)            # pad-then-truncate to exactly 2
    minor = (whole.cast("decimal(38,0)") * F.lit(100) + frac2.cast("decimal(38,0)")).cast("bigint")
    return (F.when(s.isNull() | (s == '') | (s == '0'), F.lit(0).cast("bigint"))
             .when(s.rlike(r'^\d+(\.\d+)?$'), minor)
             .otherwise(F.lit(None).cast("bigint")))   # invalid (TS throws) → NULL → quarantined below

# ── resolveBounces: prefer bounces; else round(bounceRate*sessions); else NULL ──────────
def _bounces(braw):
    direct = _count(F.get_json_object(braw, '$.bounces'))
    rate = F.get_json_object(braw, '$.bounceRate').cast("double")
    sess = F.get_json_object(braw, '$.sessions').cast("double")
    computed = F.round(rate * sess).cast("bigint")   # Math.round == Spark round HALF_UP for non-negative
    return F.when(direct.isNotNull(), direct).when(rate.isNotNull() & sess.isNotNull(), computed) \
            .otherwise(F.lit(None).cast("bigint"))

# ── hashToUuidShaped(input) port: sha256→16B, v5 nibble + RFC-4122 variant, 8-4-4-4-12 ──
def _uuid_shaped(seed):
    h = F.substring(F.sha2(seed, 256), 1, 32)                       # first 16 bytes as 32 hex chars
    var = F.lower(F.hex(F.pmod(F.conv(F.substring(h, 17, 1), 16, 10), F.lit(4)) + F.lit(8)))  # (n&3)|8
    return F.concat(
        F.substring(h, 1, 8), F.lit('-'),
        F.substring(h, 9, 4), F.lit('-'),
        F.lit('5'), F.substring(h, 14, 3), F.lit('-'),              # byte[6] hi-nibble = version 5
        var, F.substring(h, 18, 3), F.lit('-'),                     # byte[8] hi-nibble = variant
        F.substring(h, 21, 12))

    s=lower(trim(c)); return (F.when(s.contains("high"),"high").when(s.contains("medium")|s.contains("med"),"medium")
        .when(s.contains("low"),"low").when(s.contains("control"),"control").otherwise("unknown"))

def build(spark):
    salts = broadcast(_load_salts(spark))
    # ── AWB lane ──
    awb = spark.table(AWB_RAW)
    nс = norm_status(col("provider.status"))

        .join(salts, "brand_id", "left")

            trim(col("provider.status")).alias("status"), col("tclass").alias("terminal_class"),
            (col("tclass")!="none").alias("is_terminal"), pay(col("provider.payment_method")).alias("payment_method"),
            trim(col("provider.pincode")).alias("pincode"), col("scat").alias("status_changed_at"), col("scat").alias("occurred_at")))
        .withColumn("occurred_at", F.to_timestamp(col("scat"))))
    # ── RTO lane ──
    rto = spark.table(RTO_RAW)
    r = (rto.where(col("brand_id").isNotNull() & col("provider.order_id").isNotNull() & (col("provider.order_id")!=""))
        .withColumn("event_type", lit("gokwik.rto_predict.v1"))
        .withColumn("event_id", uuid_shaped(F.concat_ws(":", col("brand_id"), col("provider.order_id"), coalesce(col("provider.request_id"),lit("")), lit("gokwik.rto_predict.v1"))))
        .withColumn("props", struct(
            lit("gokwik").alias("source"), col("data_source").alias("data_source"), col("provider.order_id").alias("order_id"),
            col("provider.request_id").alias("request_id"), risk(col("provider.risk_flag")).alias("risk_flag"),
            trim(col("provider.risk_flag")).alias("risk_flag_raw"), col("provider.risk_reason").alias("risk_reason"),
            col("provider.occurred_at").alias("occurred_at")))
        .withColumn("occurred_at", F.to_timestamp(col("provider.occurred_at"))))
    def canon(df):
        return df.select(col("event_id"), col("brand_id"), col("occurred_at"),
            coalesce(F.to_timestamp(col("fetched_at")), current_timestamp()).alias("ingested_at"),
            lit("brain.collector.event.v1").alias("schema_name"), lit(1).alias("schema_version"),
            col("event_type"), lit(None).cast("string").alias("correlation_id"),
            F.concat(col("brand_id"), lit(":"), col("event_id")).alias("partition_key"),
            to_json(struct(col("event_type").alias("event_name"), col("occurred_at"), col("props").alias("properties"))).alias("payload"))
    out = canon(a).unionByName(canon(r))   # + webhook lane (same pattern, allowlist properties)
    create_iceberg_table(spark, SILVER_NAMESPACE, TARGET.split(".")[-1],
        \"\"\"event_id string NOT NULL, brand_id string NOT NULL, occurred_at timestamp NOT NULL, ingested_at timestamp NOT NULL,
           schema_name string NOT NULL, schema_version int NOT NULL, event_type string NOT NULL, correlation_id string,
           partition_key string NOT NULL, payload string NOT NULL\"\"\", partitioned_by="bucket(256, brand_id), days(occurred_at)")
    out.createOrReplaceTempView("_gk")
    spark.sql(f"MERGE INTO {TARGET} t USING (SELECT * FROM (SELECT *, row_number() OVER (PARTITION BY brand_id,event_id ORDER BY ingested_at DESC) rn FROM _gk) WHERE rn=1) s ON t.brand_id=s.brand_id AND t.event_id=s.event_id WHEN MATCHED THEN UPDATE SET * WHEN NOT MATCHED THEN INSERT *")
    return TARGET, spark.table(TARGET).count()

_RTO = {

def _normalize_status(raw: str) -> str:
    import re
    s = (raw or "").strip().lower()
    s = re.sub(r"[_-]+", " ", s)

@udf(StringType())

    if raw_awb is None or str(raw_awb).strip() == "" or not salt_hex:
        return None
    norm = str(raw_awb).strip().lower()
    h = hashlib.sha256()
    h.update(bytes.fromhex(salt_hex))
    h.update(norm.encode("utf-8"))
    return h.hexdigest()


@udf(StringType())

    seed = f"{brand_id}:{awb_seed}:{status}:{status_changed_at}:shiprocket.shipment_status.v1"
    digest = hashlib.sha256(seed.encode("utf-8")).digest()
    b = bytearray(digest[:16])
    b[6] = (b[6] & 0x0F) | 0x50  # version 5
    b[8] = (b[8] & 0x3F) | 0x80  # RFC-4122 variant
    hx = b.hex()
    return f"{hx[0:8]}-{hx[8:12]}-{hx[12:16]}-{hx[16:20]}-{hx[20:32]}"


@udf(StringType())
def resolve_payment_method(raw):
    s = (raw or "").strip().lower()
    if s in ("cod", "cash_on_delivery", "cash on delivery"):
        return "cod"
    if s in ("prepaid", "online", "paid"):
        return "prepaid"
    return None


def _load_salts(spark: SparkSession) -> DataFrame:
    return (
        spark.read.format("jdbc")
        .option("url", PG_JDBC_URL).option("user", PG_USER).option("password", PG_PASSWORD)
        .option("driver", "org.postgresql.Driver").option("query", SALT_QUERY).load()
    )


def _build(spark: SparkSession) -> DataFrame:
    raw = spark.table(RAW_TABLE)
    # Provider keys land RAW with edge-variant names — coalesce them HERE (the coercion that used to
    # live in ShiprocketWebhookStrategy / shiprocket-client now lives in governed Spark Silver).
    src = raw.select(
        col("brand_id").cast("string").alias("brand_id"),  # server-trusted envelope (MT-1)
        coalesce(col("order_id"), col("channel_order_id")).cast("string").alias("order_id_raw"),
        coalesce(col("awb"), col("awb_code")).cast("string").alias("awb_raw"),
        coalesce(col("current_status"), col("status")).cast("string").alias("status_raw"),
        coalesce(col("status_date"), col("status_changed_at"),
                 col("fetched_at")).cast("string").alias("status_changed_at_raw"),
        coalesce(col("payment_method")).cast("string").alias("payment_method_raw"),
        coalesce(col("pincode"), col("pickup_pincode")).cast("string").alias("pincode_raw"),
        coalesce(col("courier_name"), col("courier")).cast("string").alias("courier_raw"),
        coalesce(col("data_source"), lit("real")).alias("data_source"),
        col("_kafka_timestamp").alias("_kafka_ts"),
        col("_kafka_offset").alias("_kafka_offset"),
    )

    typed = (
        src
        .withColumn("order_id", trim(col("order_id_raw")))
        .withColumn("status", trim(col("status_raw")))
        # status_changed_at: keep verbatim ISO string (mart col is varchar), fallback to kafka ts.
        .withColumn("status_changed_at", coalesce(trim(col("status_changed_at_raw")),
                                                  col("_kafka_ts").cast("string")))
        .withColumn("occurred_at", col("status_changed_at").cast("timestamp"))
        .withColumn("terminal_class", classify_status(col("status")))
        .withColumn("payment_method", resolve_payment_method(col("payment_method_raw")))
        .withColumn("pincode", trim(col("pincode_raw")))
        .withColumn("courier", trim(col("courier_raw")))
        .withColumn("source", lit("shiprocket"))
        .withColumn("is_synthetic", (col("data_source") == lit("synthetic")))
    )

    salts = _load_salts(spark)
    enriched = (
        typed.join(salts, on="brand_id", how="left")
        .withColumn("awb_number_hash", hash_awb(col("awb_raw"), col("salt_hex")))
        .withColumn("event_id", event_uuid(col("brand_id"), col("awb_raw"),
                                           col("status"), col("status_changed_at")))
        .withColumn("is_terminal", col("terminal_class") != lit("none"))
        .withColumn("updated_at", current_timestamp())
    )

    keyed = enriched.where(col("order_id").isNotNull() & (col("order_id") != lit("")))

    # Replay-safe dedup: last-writer by kafka offset/occurred_at within (brand_id, event_id).
    w = Window.partitionBy("brand_id", "event_id").orderBy(col("occurred_at").asc(),
                                                           col("_kafka_offset").asc())
    deduped = (
        keyed.withColumn("_rn", row_number().over(w)).where(col("_rn") == 1)
    )

    return deduped.select(
        "brand_id", "event_id", "order_id", "source", "awb_number_hash", "status",
        "terminal_class", "is_terminal", "payment_method", "pincode", "courier",
        "status_changed_at", "occurred_at", "is_synthetic", "updated_at",
    )


def run(spark: SparkSession) -> None:
    create_iceberg_table(
        spark, SILVER_NAMESPACE, SILVER_TABLE.split(".")[-1], _COLUMNS,
        partitioned_by="bucket(256, brand_id), days(occurred_at)",
    )
    df = _build(spark)
    df.createOrReplaceTempView("sr_shipment_batch")
    spark.sql(
        f"""
        MERGE INTO {SILVER_TABLE} t
        USING (SELECT * FROM sr_shipment_batch) s
        ON t.brand_id = s.brand_id AND t.event_id = s.event_id
        WHEN MATCHED THEN UPDATE SET *
        WHEN NOT MATCHED THEN INSERT *
        """
    )
    print(f"[silver_shipment_event_shiprocket] MERGE done — {SILVER_TABLE} "
          f"rows={spark.table(SILVER_TABLE).count()}", flush=True)


def main() -> None:
    spark = build_spark("silver-shipment-event-shiprocket")
    spark.sparkContext.setLogLevel("WARN")
    run(spark)


if __name__ == "__main__":
    main()

    coalesce, col, concat, encode, from_unixtime, lit, lower, sha2, trim, unhex,
    when, get_json_object, to_timestamp, regexp_extract,
)

TABLE = "silver_settlement"
BRONZE_NS = os.environ.get("BRONZE_NAMESPACE", "brain_bronze")
RAW_TABLE = f"{CATALOG}.{BRONZE_NS}.razorpay_raw"
# get_brand_identity_salt (SECURITY DEFINER, KMS-unwraps tenancy.brand_identity_salt/0109) via JDBC.
PG_JDBC = os.environ.get("BRONZE_PG_JDBC_URL", "jdbc:postgresql://postgres:5432/brain")
PG_USER = os.environ.get("BRONZE_PG_USER", "brain")
PG_PW   = os.environ.get("BRONZE_PG_PASSWORD", "brain")

COLUMNS_SQL = """
  brand_id string NOT NULL, event_id string NOT NULL, source string, entity_type string,
  settlement_id string, order_id string, payment_id_hash string, utr_hash string,
  refund_id_hash string, dispute_id_hash string, dispute_lifecycle string, dispute_direction string,
  status string, amount_minor bigint, fee_minor bigint, tax_minor bigint, currency_code string,
  reconciliation_type string, settlement_at timestamp, occurred_at timestamp NOT NULL,
  ingested_at timestamp NOT NULL
""".strip()

def r(path):  # raw provider field off the verbatim payload JSON string column `raw`
    return get_json_object(col("raw"), f"$.{path}")

def hash_id(raw_col, salt_col):
    # FAITHFUL port of hashRazorpayId: sha256( bytes.fromhex(saltHex) ++ utf8(value.trim().lower()) )
    norm = lower(trim(raw_col))
    return when(raw_col.isNull(), lit(None)) \
        .otherwise(sha2(concat(unhex(salt_col), encode(norm, "utf-8")), 256))

def to_iso_ts(raw_col):  # unix-seconds (all-digit) *1000 OR ISO string — mirrors toIso
    digits = regexp_extract(raw_col, "^[0-9]+$", 0)
    return when(digits != "", to_timestamp(from_unixtime(raw_col.cast("long")))) \
        .otherwise(to_timestamp(raw_col))

def build(spark: SparkSession):
    fqtn = ensure_silver_table(spark, TABLE, COLUMNS_SQL,
                               partitioned_by="bucket(256, brand_id), days(occurred_at)")
    salts = (spark.read.format("jdbc").option("url", PG_JDBC).option("user", PG_USER)
             .option("password", PG_PW).option("driver", "org.postgresql.Driver")
             .option("query", "SELECT brand_id::text AS brand_id, salt_hex FROM get_brand_identity_salt_all()")
             .load())  # one (brand_id, 64-hex salt) row per brand; KMS unwrap happens inside PG

    raw = (spark.table(RAW_TABLE)
           .select(col("brand_id"), col("event_id"),
                   col("resource"), col("razorpay_event"),
                   col("raw"),  # verbatim provider JSON string (recon item OR webhook entity)
                   coalesce(to_timestamp(col("_kafka_timestamp")), col("fetched_at")).alias("ingested_at")))
    # C4: only allowlisted paths are read below; card.* is NEVER selected → dropped, PCI boundary held.
    j = raw.join(salts.hint("broadcast"), "brand_id", "left")
    entity = coalesce(r("entity_type"), lit("payment"))
    settled = to_iso_ts(r("settled_at"))
    staged = j.select(
        col("brand_id"), col("event_id"), lit("razorpay").alias("source"),
        entity.alias("entity_type"),
        r("settlement_id").alias("settlement_id"), r("order_id").alias("order_id"),
        hash_id(r("payment_id"), col("salt_hex")).alias("payment_id_hash"),
        hash_id(r("utr"), col("salt_hex")).alias("utr_hash"),
        hash_id(r("id"), col("salt_hex")).alias("refund_id_hash"),     # rfnd_* on refund entities
        lit(None).cast("string").alias("dispute_id_hash"),
        lit(None).cast("string").alias("dispute_lifecycle"),
        lit(None).cast("string").alias("dispute_direction"),
        r("status").alias("status"),
        coalesce(r("amount").cast("bigint"), lit(0)).alias("amount_minor"),
        coalesce(r("fee").cast("bigint"),    lit(0)).alias("fee_minor"),
        coalesce(r("tax").cast("bigint"),    lit(0)).alias("tax_minor"),
        coalesce(upper(trim(r("currency"))), lit("INR")).alias("currency_code"),
        when((entity == "adjustment") | r("payment_id").isNull(), lit("brand_level"))
            .otherwise(lit("per_order")).alias("reconciliation_type"),
        settled.alias("settlement_at"),
        coalesce(settled, to_iso_ts(r("created_at"))).alias("occurred_at"),
        col("ingested_at"),
    ).where(col("event_id").isNotNull() & col("brand_id").isNotNull()
            & col("occurred_at").isNotNull())
    merge_on_pk(spark, fqtn, staged, ["brand_id", "event_id"],
                order_by_desc=["ingested_at", "occurred_at"])
    return fqtn, spark.table(fqtn).count()

if __name__ == "__main__":
    run_job("silver-settlement", build)
