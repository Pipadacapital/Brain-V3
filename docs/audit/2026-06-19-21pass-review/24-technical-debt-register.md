# Technical Debt Register (2026-06-19)

> Principal-level audit of the Brain AI-Native Commerce OS monorepo. Every row is anchored to a real `file:line` opened during the audit. Interest = the compounding cost of NOT paying the debt down. Deliberate debt = a conscious phase/scope deferral or pragmatic shortcut; Accidental debt = drift, copy-paste, or a guard that silently never fires.

## Legend

- **Type**: `design` (architecture / boundary / DDD), `test`, `infra` (deployment / IaC / CI), `security` (isolation / enforcement backstop), `data` (schema / migration / dependency graph).
- **Nature**: **Deliberate** (knowingly deferred or pragmatically chosen) vs **Accidental** (drift / copy-paste / inert guard nobody noticed).
- **Effort**: S ≤ 0.5d, M ≤ 2d, L ≤ 1wk, XL > 1wk.
- **Priority**: P1 (pay down this cycle), P2 (next cycle), P3 (backlog / opportunistic).

## Distribution

**By type:** design 10 · security/isolation 6 (several rows are dual-typed) · infra 4 · data 4 · test 4.

**By nature:** Accidental 13 · Deliberate 6. The deliberate debt is concentrated in **phase deferrals** (ARC-4 identity control-plane, ARC-5 billing/recommendation/job-orchestration, ARC-6 Argo, ARC-8 IaC) and one **pragmatic shortcut** (ARC-3 Postgres spool vs ADR-003 disk WAL). The accidental debt is dominated by **copy-paste duplication** (CQ-1/2/3/4, DP-1) and **inert enforcement guards** (RS-1, ARC-1, ARC-2 — three independent ways the metric-engine/boundary fences silently never fire).

**Highest-interest cluster:** the three inert boundary guards (RS-1, ARC-1, ARC-2) jointly remove the *only* CI backstop protecting the brand-scoped metric-engine isolation seam — a multi-tenant leakage surface with zero alerting. All three are P1 and collectively < 1 day of effort.

---

## Verdict

**19 distinct debt items; headline: the multi-tenant isolation CI backstop is effectively non-functional.** Three independent, low-effort root causes (RS-1 descriptor ordering, ARC-1 absolute-glob mismatch, ARC-2 missing `@brain/*` resolver) leave the metric-engine fence and the `workspace-access/internal` reach-around rule silently inert — guards that exist in config but never fire, with no runtime alert. These are the cheapest, highest-leverage fixes in the register (all P1, all ≤ M effort) and should be paid down this cycle. The largest *structural* debt is deliberate phase deferral: four bounded contexts (identity control-plane, billing, recommendation, job-orchestration) plus the Argo/Helm/Dockerfile deployment substrate are `.gitkeep` stubs, meaning Brain currently has no GitOps path to production, no platform revenue loop, and no decision-intelligence output for any tenant — acceptable only if explicitly phase-gated, which today they are **not** (no machine-readable readiness marker in code).
