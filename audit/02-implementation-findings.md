# Phase 2 — Implementation Audit: Findings Register (AuditP2)

**Date:** 2026-07-11 (continues `01-architecture-gaps.md`; AUD-* numbering continues with the `AUD-IMPL-*` prefix)
**Scope:** Implementation deep-dive across four dimensions: (A) code quality & architecture hygiene, (B) tests & correctness of the money/identity/medallion core, (C) dead/stale code & orphaned surface, (D) performance & resource posture. Method: read-only — repo deep-reads, local execution of lint/unit/pure-Python guard suites, and read-only `kubectl`/`aws` inspection of the live `brain-prod` EKS cluster (acct 380254378136, ap-south-1). No mutations were performed; any check that would require one is tagged HYPOTHESIS.
**Evidence tags:** MEASURED = verified by execution or direct read of code/live system today. HYPOTHESIS = statically derived, needs a live/mutating verification.

---

## 0. Executive summary

The implementation core is **strong where it counts**: hexagonal boundaries are mechanically enforced (eslint-plugin-boundaries passes clean), base tsconfig is strict + `noUncheckedIndexedAccess`, production code has zero `@ts-ignore`, and the attribution/order-economics money math is exact-integer and closed-sum in both TypeScript and its Python port — verified by execution (metric-engine 400/400 incl. parity oracle, money 10/10, all pure-Python economics/identity golden tests green). The 26 local TS unit-test failures are all missing-Postgres/env artifacts, not logic bugs.

The decision-relevant gaps are **systemic, not arithmetic**:

1. **Prod-operational (HIGH):** Spark test files ship inside the prod image and are matched by the v4-silver/v4-gold CronWorkflow submit globs — one shells out to a pytest that isn't installed, which should abort v4-gold mid-run under `set -euo pipefail` once pods actually start (currently masked by the AUD-LIVE-3 ServiceAccount bring-up issue). The ADR-0006 D4 raw-PII retention job has **no prod CronWorkflow at all**. The Kafka Connect Bronze landing worker (sole landing writer per ADR-0010) measurably idles at **79 % of its memory limit**.
2. **Correctness-on-replay (MEDIUM, HYPOTHESIS):** the shared MERGE discipline has three idempotency blind spots — no staleness guard on Silver `WHEN MATCHED UPDATE` (historical replay regresses entities), no not-matched-by-source delete on Gold full-recompute rollups (the exact orphan class that previously inflated `gold_revenue_ledger` by +₹0.98 Cr), and an insert-only attribution credit ledger that can break per-order closed-sum parity when journeys mutate under re-stitch. Stitch v2's event-time watermark silently skips late/backfilled touches.
3. **Guard-suite blind spot (MEDIUM):** the `db/iceberg/spark` Python guard/parity suite runs in **no CI workflow** and two of its 19 guards are currently red and unnoticed (broadens AUD-ID-10).
4. **Scale posture (HIGH/MEDIUM, HYPOTHESIS):** the forever-retained, unpartitioned `collector_events_connect` table forces full-scan + JSON-parse reads for both the hourly Silver incremental and the uncached BFF health endpoints — cost grows linearly with all event history.

**Counts:** 4 HIGH · 13 MEDIUM · 12 LOW actionable findings, plus 3 verified-strong (no-action) assurance results.
