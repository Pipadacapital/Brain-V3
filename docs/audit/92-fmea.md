# FMEA — Failure Mode & Effects Analysis

Scoring 1–10. **RPN = Severity × Likelihood × (11 − Detectability_as_built)**, i.e. *poor* detectability raises RPN. Detectability column shows the as-built detectability score (1 = undetectable until customer reports, 10 = caught instantly by an existing gate/alarm). Ordered by RPN descending.

**Highest-RPN cluster (the "fix-first six"):** F3 (blind), F6 (double-count), F7 (no attribution), F1 (cross-tenant), F2 (read outage), F4 (event loss) — all carry RPN ≥ 486, all driven by *near-undetectable* failure modes (Detect ≤ 3). Note F8 (cannot deploy) has a lower RPN only because it is *maximally detectable* (the build fails loudly) — it is still an absolute blocker, just not a silent one.
